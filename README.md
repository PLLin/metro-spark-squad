## 🚀 本地啟動與預覽步驟

若您想在本地執行本專案，您可以直接透過以下步驟啟動：

### 1. 安裝依賴與啟動開發伺服器
在專案根目錄下，開啟終端機並執行以下指令：
```bash
# 安裝相關打包依賴
npm install

# 啟動本地 Vite 開發伺服器
npm run dev
```

### 2. 電腦瀏覽器預覽
啟動後，在電腦瀏覽器中打開終端機輸出的網址（預設通常為 `http://localhost:5173`）。
您將看到**精美的電腦版雙欄介面**：
* **左側**：展示本提案的商業核心機制與即時試用流程控制台（共 4 大核心步驟與日常模擬）。
* **右側**：一個完全可互動的 **iPhone 15 Pro 模擬器**，您可以在上面像操作真實 App 一樣點擊、滑動、切換頁籤。

### 3. 手機瀏覽器預覽（體驗真實 App 質感）
本原型支援手機自適應佈局。當直接使用手機瀏覽時，會**自動隱藏電腦版外框與左側控制台，直接滿版呈現手機 App 介面**，操作感與原生 App 無異。

* **步驟**：
  1. 確保您的**手機**與**電腦**連接到**同一個 Wi-Fi 網路**。
  2. 查看執行 `npm run dev` 後終端機所輸出的 `Network` 網址，在手機瀏覽器中輸入該網址，即可在手機上體驗滿版 App！

## 🎯 原型核心機制與互動測試引導

請依序點擊左側控制台的試用按鈕，或在右側手機模擬器內進行以下操作，體驗完整的點數眾籌與 AI 任務流程：

### 1. 👥 智慧社群金庫：連結戰友 (邀請好友)
* **操作**：點擊左側控制台的 **「步驟 01：邀請好友加入」**（或在手機「金庫」頁籤中點擊「+ 邀請成員加入金庫」）。
* **現象**：App 內會彈出邀請視窗並自動帶入好友「王大同」的資料。確認送出後，王大同將攜帶初始點數成功連結加入金庫！金庫總點數會即時播放動畫遞增，並寫入最近入點交易日誌。

### 2. 🚦 選擇乘車隊員與模擬出站 (解鎖任務)
* **操作**：在手機「首頁」的「通勤軌跡模擬」中，終點站選擇「台大醫院 (R09)」（或「中山 (R11 / G14)」）。
* **隊員勾選**：在「選擇同行乘車隊員」清單中（採用現代 Filter Chips 藥丸設計），點選勾選參與此次出行的隊員（如：我、楊小芸），最後點擊 **「模擬乘車出站」**（或點擊左側控制台的 **「步驟 02：模擬雙人乘車」**）。
* **防錯機制**：若您進行中的任務目標站與您選擇的乘車終點站不符，系統會發出警報並中止，具有極高模擬真實度。
* **解鎖現象**：
  * 手機頂部 **「動態島 (Dynamic Island)」** 變寬播放分析動畫。
  * 分析成功後，系統動態生成限時任務，下方「任務」頁籤亮起紅點，並自動切換至「任務」分頁。
  * 隊員狀態清單中，**有勾選乘車的隊員狀態會顯示為「已出站 ✓」，未勾選的隊員則會顯示為「未參與」**。

### 3. 📸 Gemini Vision 視覺審核 (上傳收據驗證)
* **操作**：在任務分頁的步驟三「上傳收據或消費拍照」中，點擊相機卡片上傳收據（或點擊左側控制台的 **「步驟 03：模擬收據驗證」**）。
* **現象**：
  * 畫面進入 **Vertex AI 與 Gemini 多模態視覺模型掃描動畫**。
  * 系統雷射線上下移動，即時輸出 AI 語意辨識明細（自動識別出目標商圈合作店家的發票明細，比對出站軌跡與金額無誤）。
  * 審核成功後，點數將**動態累加注入金庫**，並計入個人與隊員的貢獻排行榜。

### 4. 💰 金庫滿額分帳結算與環境重置 (完成閉環)
* **操作**：當金庫眾籌點數成功突破 **`15,000 pt`** 時（或點擊左側控制台的 **「步驟 04：金庫滿額結算」** 模擬累加點數至 15,050 pt 達標）。
* **現象**：
  * 系統自動彈出 **「捷運動態金庫達標！」分帳結算視窗**。
  * 根據當前金庫的實際人數，**動態計算出均等平分 (Equal Split) 後每人可領回的點數明細**（例如 15,050 pt / 3人 ➔ 每人分得 `+5,016 pt`）。
  * 點擊「確認分帳結算並重置 Demo」後，系統會播放派發點數動畫，並**將所有金庫數據、隊員名單、任務步驟重置為最一開始的初始狀態**，以便重新進行下一輪演示！
